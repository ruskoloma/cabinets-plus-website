# Cabinets Plus Website

Marketing site and product catalog for Cabinets Plus, built with Next.js and TinaCMS.

## Stack

- Next.js App Router
- React 19
- TinaCMS for editable content
- Tailwind CSS 4
- Local markdown/JSON content with Tina query fallbacks

## Local Development

```bash
npm install
npm run dev
```

Open `http://localhost:3000` for the site. Tina admin assets are built into `public/admin`.

## Scripts

```bash
npm run dev
npm run build
npm run start
npm run lint
```

## Content Structure

- `content/pages`: generic marketing pages
- `content/services`: service landing pages
- `content/cabinets`: cabinet catalog entries
- `content/countertops`: countertop catalog entries
- `content/projects`: gallery/project detail entries
- `content/global`: shared site settings and catalog configuration

## Environment Variables

The app can run with local content only, but these variables are used in different parts of the repo:

- `NEXT_PUBLIC_SITE_URL`: canonical production site URL for metadata, robots, and sitemap generation. Defaults to `https://www.spokanecabinetsplus.com` if unset.
- `NEXT_PUBLIC_WEB3FORMS_ACCESS_KEY`: Web3Forms access key used by the browser contact form. Web3Forms documents this key as safe to include in form markup.
- `NEXT_PUBLIC_TINA_CLIENT_ID`
- `TINA_TOKEN`
- `S3_BUCKET`
- `S3_REGION`
- `S3_ACCESS_KEY`
- `S3_SECRET_KEY`
- `S3_CDN_URL`
- `S3_MEDIA_ROOT`
- `STRAPI_TOKEN` or `STRAPI_API_TOKEN` for import scripts
- `OPENROUTER_API_KEY` for content-enrichment scripts

## Notes

- Route loaders first try Tina queries and then fall back to local files in `content/`.
- Import and migration utilities live in `scripts/`.
- Shared layout configuration is stored in `content/global/header.json`, `content/global/footer.json`, and `content/global/general.json`.
- The Tina S3 media route prefers explicit `S3_BUCKET` and `S3_REGION`, but can also derive them from `S3_CDN_URL` when the CDN URL uses the standard virtual-host S3 form like `https://bucket.s3.us-west-2.amazonaws.com`.
- `robots.txt` and `sitemap.xml` are generated by Next metadata routes at `/robots.txt` and `/sitemap.xml`; do not commit generated copies into `public/`.
